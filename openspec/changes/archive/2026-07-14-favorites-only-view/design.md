## Context

V1 已具备：角色侧栏、预览 + 变体缩略图、应用壁纸、收藏开关与本地持久化、换一张（随机角色）、‹ › 切角色。PRD P1 已列出 Favorites-only browse view；本变更将其落地为独立「收藏夹」模式（方案 C），而非角色列表过滤。

首次落地后用户反馈：只看收藏往往只剩一张；缺少【取消收藏】按钮文案。根因包括：浏览器/Vite 预览路径下 `toggleFavorite` 曾返回单元素列表（不累加），以及按钮用「已收藏」兼作取消，语义不清。本设计更新闭合这些缺口。

约束（与现有产品一致）：仅【应用】可改系统壁纸；壁纸 IO 在 Rust；UI 文案来自 CONTENT.md；无账号/云。

## Goals / Non-Goals

**Goals:**

- 提供【只看收藏】开关，进入以「已收藏立绘」为条目的浏览模式
- 多张收藏必须全部出现在收藏夹列表（条数 = 收藏张数）
- 范围内导航（‹ ›、换一张）只作用于当前收藏列表
- 空收藏不可进入；收藏夹内减至 0 自动退回全部
- 启动始终默认「看全部」
- 收藏操作文案：【收藏】 / 【取消收藏】

**Non-Goals:**

- 收藏拖拽排序、分组、备注、搜索
- 记住上次是否开启只看收藏
- 定时换壁纸、托盘、云同步
- 改变收藏持久化格式的对外语义（除非实现必须做兼容迁移，且对用户不可见）

## Decisions

### 1. 浏览模式：收藏夹（C），不是角色过滤

- **选择**：侧栏在开启时展示「每张已收藏立绘一条」；关闭时恢复角色列表。
- **备选**：A 只显示有收藏的角色 / B 角色 + 仅收藏变体 — 已否决，用户明确选 C。
- **理由**：收藏对象是具体图，不是「这个角色」。

### 2. 一条 = 一张已收藏立绘；多张必须累加

- 同一角色多张收藏 → 侧栏多条；标签含角色名与变体标识。
- 排序：角色 ID 升序，同角色内按变体序号升序（稳定、可预期；不做「最近收藏」）。
- **任何** 收藏读写路径（Tauri 真持久化或 Vite 开发 mock）在连续收藏 N 张后，列表与内存状态都必须保留 N 条；禁止「只返回最后一张」的 mock。

### 3. 预览：只显示选中这一张

- 开启只看收藏时，隐藏（或不渲染）「该角色全部变体」缩略图条。
- 【应用】【收藏】【取消收藏】仅针对当前选中这一张。

### 4. 空收藏与减至空

- 收藏数为 0 时点击【只看收藏】：不进入；提示「请至少收藏一张图片」。
- 已在收藏夹内时取消收藏导致列表为空：自动关闭只看收藏，回到全部角色浏览（可选轻提示，非必须）。

### 5. 导航范围

- 开启时：‹ › 在收藏条目间移动；【换一张】在收藏条目中随机（不少于 2 条时避免连续重复同一张为佳，非硬性验收）。
- 关闭时：行为与 V1 完全一致。

### 6. 会话默认

- `favoritesOnly`（或等价 UI 状态）不写入持久化；每次启动为关。

### 7. 文案

- 开关标签：【只看收藏】
- 空拦截提示：请至少收藏一张图片
- 未收藏按钮：【收藏】
- 已收藏按钮：【取消收藏】（用户确认；不用「已收藏」作按钮文案）
- 字符串进 CONTENT.md，代码不硬编码中文。

### 8. 验收强度

- 静态源码扫描可作为辅助，但不足以宣称完成。
- 验收脚本至少模拟/断言：连续加入 ≥2 个不同 filename 后，收藏集合长度 ≥2，且 `buildFavoritesGallery` 行数 ≥2。
- 桌面路径仍以 `%APPDATA%/touhou-wallpaper/favorites.json` 为真源。

## Risks / Trade-offs

- [侧栏语义切换] 用户短暂困惑「角色列表哪去了」→ 开关文案清晰；关闭即回全部。
- [取消最后一张] 自动退出可能突然回到角色列表 → 与「空不能进」一致，可接受；若需提示可后补。
- [与变体条习惯冲突] 收藏夹不显示变体 → 符合「盯这一张」；要看同角色其他变体须先关掉只看收藏。
- [Vite mock 与 Tauri 行为分叉] → mock 必须累加；否则只看收藏验收在浏览器里必假绿。

## Migration Plan

- 无数据迁移要求；既有 favorites 继续有效。
- 回滚：去掉开关与模式分支即可，不影响收藏文件。
