## Why

收藏变多后，用户仍只能按「全部角色」翻找，难以把已收藏的立绘当图库快速浏览。需要一个独立的「只看收藏」浏览模式，专门查看和管理已收藏的单张立绘。

首次实现后实测暴露缺口：多张收藏未正确累加（只看收藏常只剩一张）；收藏按钮用「已收藏」作文案，缺少明确的【取消收藏】。本更新把这些收进需求与验收。

## What Changes

- 新增 **【只看收藏】** 开关：开启后进入收藏夹浏览（不是简单过滤角色列表）
- 收藏夹中 **一条 = 一张已收藏的立绘**（同一角色多张收藏 = 多条）
- **连续收藏多张后进入只看收藏，侧栏条数必须等于已收藏张数**（不得只保留最后一张）
- 选中某条后 **只预览这一张**（不展示该角色全部变体缩略图）
- 开启时，**【换一张】** 与 **上一个 / 下一个** 仅在当前收藏列表范围内工作
- 无收藏时 **不能进入**；点击开关提示「请至少收藏一张图片」
- 在收藏夹内取消收藏至 0 张时 **自动退出**「只看收藏」，回到全部角色浏览
- 每次启动应用 **默认看全部**（不记住上次是否开启只看收藏）
- 关闭开关后恢复现有的全部角色浏览行为
- ActionBar 收藏操作文案：**未收藏显示【收藏】；已收藏显示【取消收藏】**（不再用「已收藏」作按钮文案）；仅当前图已收藏时可执行取消
- 不改变：必须点【应用】才设桌面壁纸；收藏仍本地持久化（桌面应用路径）
- 验收须覆盖多张收藏累加与【取消收藏】文案，不能仅靠静态源码扫描宣称完成

## Capabilities

### New Capabilities

- `favorites-gallery`: 独立收藏夹浏览模式（开关、列表条目、预览、范围内导航、空收藏拦截、取消至空自动退出、多张累加可见、收藏/取消收藏文案）

### Modified Capabilities

- （无既有 OpenSpec capability；本变更引入新能力。现有「收藏开关 / 持久化」行为保持并强化累加正确性，仅在浏览模式上叠加。）

## Impact

- 前端：侧栏在「只看收藏」下改为收藏立绘列表；预览区隐藏变体条；ActionBar 导航与换一张范围受限；开关与提示文案；收藏按钮文案【收藏】/【取消收藏】
- 开发预览（非 Tauri）若提供收藏：必须累加多张，禁止「每次只返回当前一张」的假实现误导只看收藏
- 文案：`CONTENT.md` 增加开关、空提示、【取消收藏】等字符串
- 产品文档：`PRD.md` P1「Favorites-only browse view」落地为本变更验收
- 后端收藏读写 API 预期复用，不新增云/账号能力
- 明确不做：收藏排序拖拽、分组、搜索、记住开关状态、定时换壁纸
